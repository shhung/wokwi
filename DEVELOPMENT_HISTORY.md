## 2026-05-18: 技術架構與開發規範調整

### 使用者需求 (Prompt)
- 分析 `readme.md` 需求並提出討論。
- 1. 先以 bare-metal 方式實作，Serial、LOG 及基礎系統功能可使用 Arduino API。
- 2. 可以操控 I2C 控制器。
- 3. 用簡單的 Super loop，盡量精簡程式。
- 4. 電路圖上看起來並沒有轉接板（針對 LCD2004）。
- 5. 程式碼撰寫在 `sketch.ino`。
- 6. 每次對話完做一次 commit 紀錄。
- 7. 針對提出的 prompt 也做紀錄。

### 關鍵決策
- **實作層級**：底層驅動（I2C, 1-Wire, LCD, RTC）採用 **Bare-metal** 直接操作暫存器的方式。
- **輔助工具**：`Serial` 與基礎系統功能沿用 **Arduino API**。
- **任務排程**：使用 **Super Loop** 搭配 `millis()` 進行非阻塞定時。
- **LCD 驅動**：因位址為 `0x27`，推斷需實作 PCF8574 I2C 轉接板邏輯。

## 2026-05-18: 程式架構與資料結構設計

### 使用者需求 (Prompt)
- 目前已有基本 Serial 輸出，確認 Super loop 運作正常。
- 在接續開發前，要求說明更詳細的程式架構，包含預計如何分層、設計怎樣的資料結構。

### 關鍵決策與架構規劃
**1. 軟體分層設計 (Layering)**
為維持 Bare-metal 的極簡性同時保持可維護性，程式將分為三層：
- **硬體抽象層 (HAL/Bus Layer)**:
    - 負責最底層的時序與暫存器操作。
- **裝置驅動層 (Device Driver Layer)**:
    - 利用 Bus Layer 提供的 API 與具體感測器/顯示器溝通。
- **應用邏輯層 (Application Layer)**:
    - `setup()` 與 `loop()` 所在，負責任務排程。

## 2026-05-18: 將 I2C 重構為 Bit-Banging (軟體模擬)

### 使用者需求 (Prompt)
- 回報 DHT22 已經成功讀取數值，但時間依然無變化且 LCD 沒有畫面。

### 關鍵決策
- **重構方案**：將整個 I2C 匯流排層重寫為 **Bit-Banging (純軟體模擬)**，以繞過硬體控制器在模擬器中的不穩定因素。

---

## 2026-05-18: 系統穩定性修復與 LCD 位址偵測

### 使用者需求 (Prompt)
- LCD 依舊無畫面。
- RTC 時間無變化（停留在 00:00:00）。

### 關鍵決策
- **RTC 修復**：實作 `ds1307_init` 並清除 Clock Halt (CH) 位元。
- **LCD 位址偵測**：加入 `0x27` 與 `0x3F` 的自動掃描。

---

## 2026-05-19: 修復 I2C 掃描、RTC 初始化與 LCD 顯示邏輯 (第一階段)

### 使用者需求 (Prompt)
- 目前 serial 有持續輸出有溫溼度，但數值都沒變不確定是否正常。
- 時間依然沒有變化 (2000/00/00 00:00:00)。
- LCD 有畫面但不正常。

### 關鍵決策
- **I2C 匯流排修正**：修復掃描邏輯，增加 `I2C_DELAY()`。
- **RTC 初始化**：嘗試在偵測到 CH 位元時寫入預設時間。
- **LCD 顯示優化**：調整 4-bit 初始化序列與螢幕排版。

---

## 2026-05-19: 深入修復 RTC 離線與 Serial/LCD 格式異常

### 使用者需求 (Prompt)
- 狀況變得更糟了，LCD 仍然異常。
- Serial 出現 `RTC: ERR`，且格式不符。
- 紀錄順序應為由上往下 (由舊到新)。

### 關鍵決策
- **RTC 穩定性加強**：修正 `ds1307_init`，不僅檢查 CH 位元，若日期暫存器全為 0 也進行初始化。
- **Serial 格式對齊**：初步依照 `readme.md` 調整輸出。
- **LCD 初始化再強化**：增加初始化過程中的延遲。

## 2026-05-19: LCD 亂碼修復與新版格式最終對齊

### 使用者需求 (Prompt)
- 保留先前完整紀錄，將新開發歷程增加到之後。
- LCD 畫面依然有亂碼。
- 更新 readme 輸出格式。

### 關鍵決策
- **LCD 亂碼徹底根除**：
    - 移除所有在 LCD 上顯示中文字元的嘗試。HD44780 顯示晶片不支援 UTF-8/Big5 中文，強行寫入會導致指令集偏移造成滿畫面亂碼。
    - 改用英文標籤（`Time:`, `Date:`, `Temp:`, `Humd:`）進行提示。
- **新版 Readme 格式精確對齊**：
    - **Serial**：正常時輸出 `YYYY/MM/DD HH:MM:SS | XXX.X°C | XXX.X%`；失敗時分別顯示 `----/--/-- --:--:--` 或 `-`。
    - **LCD**：嚴格依照 Line 1~4 的位置更新，並在失敗時顯示對應的英文錯誤標籤（`RTC: ERR`, `DHT: ERR`）。
- **Git 流程強化**：嚴格執行每次對話後的 commit 紀錄，確保開發軌跡完整。
